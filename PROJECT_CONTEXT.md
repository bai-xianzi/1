# PROJECT_CONTEXT.md

## 一、项目定位

本项目是面向个人长期使用的A股量化投资辅助操作系统。

项目最高优先级工程方法是：先调查项目已有能力、官方SDK、行业标准和成熟开源实现；能直接复用就复用，能薄适配就不重写，确实没有可用方案时才独立开发必要的最小部分。项目既不简单拼接外部产品，也不闭门重复实现社区已经成熟的通用能力。

终极目标不是简单的K线软件、单一选股脚本或某个回测策略，而是一条完整、可追溯、可复现、可逐步演进的决策链：

```text
多源数据
→ 标准字段和数据质量
→ 市场状态
→ 系统性风险与仓位
→ 因子和模型
→ 股票排序
→ 目标组合
→ 真实A股回测
→ T日收盘报告
→ T+1集合竞价确认
→ 人工交易
→ 反馈和迭代
```

远期还包括：

- 全网应用统计证据发现、统计主张结构化、反证跟踪与候选规则治理；
- CWMS因果世界到市场仿真；
- 实验注册和模型生命周期；
- 模拟盘；
- 券商通道、订单状态机、幂等、停机和账实核对。

终极架构决定方向，当前阶段决定现在实际开发什么。

## 二、当前真实阶段

```text
CURRENT：通用数据接入骨架搭建 + 数据入库与质量验收
```

当前目标不是开发选股模型或前端，而是确保真实数据：

- 能稳定读取；
- 能保留原始语义；
- 能转换到标准字段；
- 能确认单位、日期和复权；
- 能生成质量与血缘结果；
- 能被未来所有下游模块复用。

## 四、数据架构主链

```text
交易所、厂商、API、数据库、文件、自定义数据
→ Raw原始层
→ 来源适配器与领域语义适配器
→ Canonical统一字段和标准对象
→ 数据质量、血缘、可见时间和冲突处理
→ 标准数据服务
→ 市场状态、风险、因子、模型、组合、回测和报告
```

DolphinDB是当前正式数据底座和首个真实适配器案例，但下游不能直接依赖DolphinDB来源字段。

## 五、稳定边界

当前应优先稳定：

- 复用、薄适配和自研之间的决策边界；
- 第三方组件的隔离、替换和回滚边界；
- 标准字段字典；
- 标准数据对象；
- `DataSourceAdapter`接口；
- 来源映射格式；
- 数据质量结果格式；
- 数据血缘格式；
- 时间和可见性规则；
- 复权和可执行价格原则；
- 模块输入输出。

算法、模型、阈值、报告、前端和券商接口以后可以逐步替换。

## 六、当前开发顺序

0. 对每个新能力先执行复用调查，确定直接复用、薄适配和最小自研边界；
1. 将本启动包放入正式Git仓库；
2. 确认权威文件完整且无冲突；
3. 定义最小`DataSourceAdapter`接口；
4. 定义Raw、Canonical、Semantic最小数据契约；
5. 定义来源映射、质量结果和血缘格式；
6. 实现DolphinDB日K首个适配器；
7. 用样例数据确认字段、类型、日期、单位和复权；
8. 生成日K质量验收报告；
9. 扩展到基本面和七类快照；
10. 数据门禁通过后，开始市场状态与风险仓位MVP。

- 自动券商交易；
