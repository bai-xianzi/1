# 项目长期记忆逻辑复核报告（V5审阅候选）

复核日期：2026-06-22

本轮遵循“只做必要重构、不大改既有架构”的要求。没有删除原有层级，没有改变当前开发主线，只补齐用户场景控制和几处文件一致性问题。

## 一、发现并修复的问题

1. **所有变量可控的表述存在歧义**：修正为“所有市场影响字段都应场景可寻址，但直接控制权限按变量角色分级”。
2. **直接市场冲击与‘事件不能直接跳到股票’存在表面冲突**：新增因果场景、直接市场扰动、混合和模块测试四种模式；直接冲击明确是测试外生输入。
3. **专家/LLM规则缺少批准边界**：新增候选、人工审核、APPROVED/ACTIVE流程；LLM不得自行启用规则。
4. **用户输入冲突缺少治理**：新增干预合同、冲突策略、优先级和 `CAUSAL_CONSISTENCY_BROKEN` 标记。
5. **字段同名的机器引用存在歧义**：当前字典中有 **74** 个跨对象复用的字段名。未进行破坏性改名，改为规定机器引用优先使用 `domain_code.canonical_name`。
6. **仿真覆盖容易被误解为已完成**：明确 `simulation_variables.yaml` 只有代表性样例，全量945+字段尚未完成控制权限和仿真合同审核。
7. **长期记忆与当前状态职责混杂**：增加文件权威边界；当前阶段以 `PROJECT_STATUS.md` 为准。
8. **PROJECT_STATUS旧信息**：日期更新，V0.1改为V0.2历史基础，并补充控制平面设计状态和新的阶段门槛。
9. **README文件清单位置混乱**：将CWMS和schemas恢复到权威文件列表。

## 二、本轮新增或调整

- 新增结构图中的 `6A-0 用户场景控制平面`；
- 新增 `scenario_intervention` 标准领域和 `scenario_interventions.yaml`；
- 字段字典升级为 `0.3-review` 审阅候选；
- 增加控制权限、运行模式、干预方式、传播模式、冲突策略和因果一致性枚举；
- CWMS文档增加控制平面、运行模式、规则审核、冲突治理和字段引用规则；
- Codex规则增加：不得批准LLM因果规则，不得把直接冲击冒充完整因果解释。

## 三、没有改变的内容

- 当前主线仍为通用数据接入骨架、DolphinDB适配和数据质量验收；
- 市场状态—风险仓位—策略—个股—组合主链不变；
- CWMS仍是横向验证底座，不抢占当前MVP开发顺序；
- 不建设微服务，不一次性开发完整战争、舆情、逐笔和盘口系统；
- 原有字段没有删除或破坏性改名。

## 四、仍需后续单独审阅的问题

1. 已补入最小跨资产市场通用领域，可承载加密货币、期货、商品、外汇和全球指数冲击；详细品种、交易制度和专用字段仍需后续专项补全。
2. 全量标准字段的 `scenario_addressable` 和控制权限尚未逐字段审核。
3. 因果规则样例仍少，且都处于草稿状态，不能用于正式结论。
4. `AGENTS.md` 和 `PROJECT_CONTEXT.md` 仍在项目记忆目录之外，正式仓库启动时要统一更新。
5. 字段跨对象复用是合理的，但未来代码生成器和校验器必须使用限定字段引用。

## 五、审阅重点

请重点确认：

- “所有市场影响字段可寻址，但不全部直接可改”是否符合你的意思；
- 直接市场扰动模式是否需要保留；
- LLM/专家规则必须人工批准的边界是否足够严格；
- 硬冲突默认拒绝、异常测试才允许强制运行是否合理；
- 当前不立即补齐全部跨资产字段是否符合渐进式开发原则。
