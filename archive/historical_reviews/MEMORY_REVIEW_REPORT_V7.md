# 项目长期记忆逻辑复核报告（V7）

复核日期：2026-06-22

## 一、本轮主要修正

### 1. 明确甲方与项目经理职责

- 甲方负责目标、场景、约束、优先级和验收；
- ChatGPT负责项目管理、领域抽象、字段命名、架构、开发顺序和技术取舍；
- Codex和Trae只负责执行，不拥有架构和金融语义决策权；
- 用户举例不自动等同于字段、数据库表或代码模块。

### 2. 正式确认通用事件—实体—关系—影响模型

- 0/1只表示事件或影响是否发生；
- 类型、目标、程度、时间、恢复和传播分别表达；
- 支柱产业属于国家—行业关系；
- 支柱产业受损属于事件对行业的动态影响；
- 仓库、工厂、港口等场景不再机械创建专用字段。

### 3. 修复V6包的文件完整性问题

V6包中实际遗漏了若干已经定义的机器可读知识资产。本轮已恢复并重新校验：

- `enum_definitions.yaml`
- `simulation_variables.yaml`
- `causal_rules.yaml`
- `scenario_catalog.yaml`
- `entity_exposure_schema.yaml`
- `FIELD_CHANGE_PROCESS.md`

### 4. 结构图轻量调整

- 第0层加入需求治理和领域建模评审；
- 新增横向“需求治理、领域抽象与技术决策”；
- CWMS正式编号调整为`6A-2`，与`6A-0`和`6A-1`顺序一致；
- 当前真实数据接入和质量验收主线不变。
