# 机构化量化研究生产线与券商级执行安全

## 1. 目的

本文件把大型量化机构和券商公开实践中可迁移的方法，转化为适合个人A股系统的长期工程原则。

项目不声称掌握任何机构的私有系统，也不照搬其硬件和组织规模。

## 2. 从量化机构借鉴的核心

### 2.1 研究是一条生产线

```text
问题与假设
→ 实验登记
→ 数据与版本冻结
→ 研究和回测
→ 样本外与滚动验证
→ 成本与容量
→ CWMS压力测试
→ 模拟盘
→ 有限实盘
→ 正式实盘或退役
```

### 2.2 Alpha不是最终产品

因子和模型只是原材料。最终结果还取决于：

- 市场状态；
- 风险门控；
- 行业与风格暴露；
- 组合集中度；
- 流动性；
- 成交成本；
- 容量；
- 执行可行性。

### 2.3 研究必须可复现

每次实验保存：

- 实验ID；
- 研究问题和假设；
- 数据快照和字段版本；
- 代码提交；
- 配置哈希；
- 随机种子；
- 训练、验证和测试区间；
- 成本和容量模型；
- 结果和失败原因。

### 2.4 失败实验也是资产

负面结果和失败原因必须保存，避免：

- 选择性汇报；
- 重复研究无效想法；
- 参数越调越漂亮；
- 忘记成本后失效；
- 后见之明偏差。


## 2A. 外部应用统计证据也是研究生产线输入

机构化研究不能只从内部因子想法开始，也应系统吸收全网公开的应用统计成果。

来源不限于正式论文，还包括：

- 预印本、工作论文和会议材料；
- 政府、监管、央行和国际组织统计；
- 大学、研究机构和智库报告；
- 调查、民调、自然实验和准实验；
- 行业协会、数据厂商和咨询机构白皮书；
- 杂志研究文章、学者刊物和数据新闻；
- 复现、反驳、方法评论、勘误和撤稿；
- 公开数据、代码、问卷和量表。

外部结果进入研究生产线时必须执行：

```text
发现
→ 结构化统计主张
→ 方法和因果等级审核
→ 标准字段映射
→ 反证与版本追踪
→ 本项目复现或样本外检验
→ 候选特征/规则实验登记
→ 晋级、争议、拒绝或退役
```

不能因为来源知名、统计显著或结论符合直觉就直接进入生产。

统计主张必须保存效应量、置信区间、样本、适用范围和限制；失败复现、相反结论和撤稿不得删除。

涉及人的统计规律必须以匿名、时点化、概率化画像使用，不能直接形成永久人格标签或单独决定评论真假、证券评分和交易动作。

详细治理见：

- `APPLIED_STATISTICAL_EVIDENCE_INTELLIGENCE.md`


## 3. 从券商借鉴的核心

### 3.1 策略不能直接发单

```text
策略和模型
→ 目标组合
→ 订单生成
→ 交易前风控
→ 券商网关
```

### 3.2 订单是状态机

典型状态包括：

```text
CREATED
→ PENDING_RISK
→ REJECTED / READY_TO_SEND
→ SENT
→ ACKNOWLEDGED
→ PARTIALLY_FILLED / FILLED
→ CANCEL_PENDING / CANCELLED
→ UNKNOWN
```

### 3.3 幂等防止重复下单

每次决策和订单必须具有稳定标识。网络超时后应先查询券商状态，不得盲目重发。

### 3.4 分级停机

至少支持：

- 暂停新开仓；
- 只允许卖出；
- 撤销全部活动订单；
- 停止单个策略；
- 停止单个账户；
- 停止全部自动交易。

### 3.5 账实核对

每日或关键时点核对：

- 订单；
- 成交；
- 持仓；
- 现金；
- 费用；
- 公司行为。

重大差异未解决前，默认禁止新的自动交易。

## 4. 环境和发布治理

至少区分：

```text
RESEARCH
BACKTEST
PAPER
LIVE
```

重要变更遵循：

```text
代码审查和测试
→ 历史回测
→ CWMS压力测试
→ 模拟盘
→ 有限资金
→ 正式部署
```

必须保留部署版本、配置版本、稳定回滚版本、备份和恢复演练记录。

## 5. 个人版的取舍

当前不建设：

- 微秒级低延迟；
- FPGA；
- 超大GPU集群；
- 多机房双活；
- 大规模微服务；
- 高频做市和全球多资产实盘。

当前优先建设：

- 数据正确；
- 研究可复现；
- 实验可登记；
- 风险可否决；
- 回测真实；
- 报告可执行；
- 未来订单不重复；
- 账实可核对；
- 系统可停机；
- 版本可回滚。

## 多供应商与券商SDK治理

商业数据、研究平台、组合分析、账户查询和交易执行必须独立授权、独立适配、独立激活。

具有潜在交易能力的Provider默认只能处于`REGISTERED_TARGET`或研究发现状态。没有独立风控、订单状态机、幂等键、账实核对、停机机制和有限实盘验收时，不得启用下单与撤单能力。

供应商许可证、缓存、历史存储和再分发边界必须进入验收证据。

## 第三方复用治理

第三方数据、研究和交易组件必须记录来源、版本、许可证、授权用途、安全状态和替换路径。

交易执行接口优先使用券商官方SDK，不得自行仿造下单协议。即使复用官方SDK，也必须保留独立风控、幂等、订单状态机和账实核对。
