# TASK_025A：全系统 B2C 双视图与 Skill 平台权威架构

## 1. 任务目标

基于用户已经确认的大局观，将 WJX 从“内部模块为主、局部增加页面”重构为“用户端操作结构图与系统内部执行结构图成对设计”的项目权威架构。

本任务只建立权威合同、机器Schema和开发门禁，不实现完整前端、人物全网蒸馏、工作流运行时或生产因子发布。

## 2. 核心决定

1. B2C 是作用于全部模块的用户控制平面，不是最后补上的前端；
2. 每个功能同时设计后台领域合同与B2C交互合同；
3. Skill 是开发和产品运行的一等公民；
4. 开发型与产品型Skill共享内核，但权限和发布上限隔离；
5. 人物蒸馏产出证据、时间线、机制假设和候选因子，不自动宣称确定因果；
6. 用户能够白箱查看链路、变量、公式、权重、得分贡献、代码、前置教学注释、测试和版本；
7. 字段、因子、仿真变量和规则变更从UI进入正式提案、验证和审批流程，不能直接写入生产；
8. 通用表单、图谱、工作流、代码查看、公式渲染和实验跟踪能力优先复用成熟社区实现。

## 3. 允许修改

- 项目长期记忆、总体架构、开发指导、项目上下文、启动说明和状态；
- Provider接入中心与全系统B2C关系说明；
- 新增B2C与Skill专项权威文件；
- 新增基础JSON Schema；
- 新增本任务验证脚本、测试和验收报告。

## 4. 禁止修改

- 任何DolphinDB数据库、表和数据；
- 现有Python业务逻辑；
- Provider、Canonical、Readiness、市场状态和评分算法；
- TASK_024D及其后续真实实施顺序；
- 自动交易或生产因子启用状态。

## 5. 验收标准

```text
两份专项权威文件存在
总体架构升级为V11双视图
长期记忆写入B2C、白箱、Skill和双合同原则
开发指导形成强制交互门禁
六个JSON Schema可解析
专项验证通过
专项unittest通过
compileall通过
git diff --check通过
无数据库写操作
```
