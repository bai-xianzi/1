# TASK_021D2：TASK_021B/021C跨Python版本验证器最终修复

## 根因

旧 `PYTHON_TOKEN_STREAM_V1` 把 `token.type` 数字编号写入持久摘要。Python版本变化时，部分Token编号变化：

- TASK_021B共10个文件，其中仅全注释的空 `__init__.py` 摘要不受影响，其余9个失败；
- TASK_021C共9个文件，全部失败；
- 全量测试因此准确出现18个失败。

这不是18个业务模块发生逻辑变化。

## 修复

改用 `ANNOTATED_TARGET_SOURCE_SHA256_V1`：

1. 读取已经成功迁移的目标源码；
2. 由Python文本读取统一CRLF/LF换行；
3. 对完整注释后源码计算UTF-8 SHA256；
4. 保留旧迁移报告中的源到目标语义一致性作为历史证据；
5. 不修改任何业务源文件，不连接DolphinDB。
